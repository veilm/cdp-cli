• Here’s the API I’d love:

  Config

  - cdp connect manager --port 2102 --url https://manager.rvm:9443/orders/27
    Stores the browser port + target URL under a name (manager). Keeps the session alive and
    remembers the tab.

  Eval

  - cdp eval manager "document.querySelector('.alert-box').getBoundingClientRect()"
    Runs arbitrary JS in that tab and prints JSON. Optionally add --pretty, --depth, etc.

  Helpers

  - cdp dom manager ".alert-box" → dumps outerHTML/text.
  - cdp styles manager ".alert-box" → prints computed styles or box metrics.
  - cdp screenshot manager --selector ".composer" → crops element screenshot.
  - cdp log manager "console.log(...)" → pipe console logs back (subscribe/unsubscribe).

  Workflow

  1. cdp connect … once per tab.
  2. Repeated eval/dom/styles commands without re-specifying port/tab.
  3. Optional cdp targets to list active sessions, cdp close manager to detach.

  With that, I can poke at layout details (DOM structure, styles, computed heights) and
  iterate without manual eyeballing.
